    datar.base.funcs.as_date(x, format=None, try_formats=None, optional=False, tz=0, origin=None)

    </>

    Convert an object to a datetime.date object

    See: https://rdrr.io/r/base/as.Date.html

    Parameters
    • x (DataFrame or DataFrameGroupBy) — Object that can be converted into a datetime.date object
    • format (str, optional) — If not specified, it will try try_formats one by one on the first non-NaN element, and give an error if none works. Otherwise, the processing is via strptime
    • try_formats (list of str, optional) — vector of format strings to try if format is not specified.
    • optional (bool, optional) — indicating to return NA (instead of signalling an error) if the format guessing does not succeed.
    • tz (int, integer, or timedelta, optional) — a time zone offset or a datetime.timedelta object. Note that time zone name is not supported yet.
    • origin (any, optional) — a datetime.date/datetime object, or something which can be coerced by as_date(origin, ...) to such an object.
    Returns (iterable of date)

    The datetime.date object

    datar.base.funcs.cut(x, breaks, labels=None, include_lowest=False, right=True, precision=2, ordered_result=False)

    </>

    Divides the range of x into intervals and codes the values in x according to which interval they fall. The leftmost interval corresponds to level one, the next leftmost to level two and so on.

    Parameters
    • x (iterable of int, float, complex, or number) — a numeric vector which is to be converted to a factor by cutting.
    • breaks (any) — either a numeric vector of two or more unique cut points or a single number (greater than or equal to 2) giving the number of intervals into which x is to be cut.
    • labels (iterable of any, optional) — labels for the levels of the resulting category. By default, labels are constructed using "(a,b]" interval notation. If labels = False, simple integer codes are returned instead of a factor.
    • include_lowest (bool, optional) — bool, indicating if an ‘x[i]’ equal to the lowest (or highest, for right = FALSE) ‘breaks’ value should be included.
    • right (bool, optional) — bool, indicating if the intervals should be closed on the right (and open on the left) or vice versa.
    • precision (int, optional) — integer which is used when labels are not given. It determines the precision used in formatting the break numbers. Note, this argument is different from R's API, which is dig.lab.
    • ordered_result (bool, optional) — bool, should the result be an ordered categorical?
    Returns (Categorical)

    A categorical object with the cuts

    datar.base.funcs.factor(x, levels=None, exclude=nan, ordered=False) → Categorical

    </>

    encode a vector as a factor (the terms ‘category’ and ‘enumerated type’ are also used for factors).

    If argument ordered is TRUE, the factor levels are assumed to be ordered

    Parameters
    • x (iterable of any) — a vector of data
    • levels (iterable of any, optional) — an optional vector of the unique values (as character strings) that x might have taken.
    • exclude (any, optional) — a vector of values to be excluded when forming the set of levels. This may be factor with the same level set as x or should be a character
    • ordered (bool, optional) — logical flag to determine if the levels should be regarded as ordered (in the order given).

    datar.base.funcs.context(data)

    </>

    Evaluate verbs, functions in the possibly modifying (a copy of) the original data.

    It mimic the with function in R, but you have to write it in a python way, which is using the with statement. And you have to use it with as, since we need the value returned by __enter__.

    • func — A function that is registered by pipda.register_verb or pipda.register_func.
    • *args — Arguments for func
    • **kwargs — Keyword arguments for func
    Returns (any)

    The original or modified data
